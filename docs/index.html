Key Concepts

What is Hashing?

A hash converts data of any size into a fixed-length unique fingerprint. The same input always produces the same hash, but even the tiniest change creates a completely different one. Critically, you cannot convert a hash back into the original data.

Example: An image produces hash "a3f2b8...". Change one pixel, and you get an entirely different hash like "d91e4c...".

What is Key Encryption?

Encryption scrambles data using a secret key so only authorized parties can decrypt and read it.

Example: Your camera encrypts its identity‚Äîonly the manufacturer with the decryption key can verify it's legitimate.

Why Blockchain?

Blockchain is a permanent, tamper-proof public ledger. Once data is recorded on a blockchain, it cannot be changed or deleted‚Äîeven if companies go out of business or try to censor records.

Example: When your photo's hash is recorded on the Birthmark blockchain, that record is permanent and publicly verifiable by anyone, forever. The blockchain is operated by trusted institutions like universities and archives.

1. Creating the Image Hash

Camera-Side Authentication

At the moment of image capture, a secure chip in the camera creates an unforgeable digital signature before the image is even processed.

  • Raw image hashing: A hash is created from the raw image data before any processing like white balance or compression
  • Processed image hashing: A second hash is created from the processed image that you actually save
  • Timestamp and metadata: The camera records when the photo was taken and optionally a hash of the GPS coordinates
  • Parallel processing: Authentication happens in the background while your photo is being captured‚Äîno delays or extra steps

Result: A bundle containing both image hashes, timestamp, and optional location hash‚Äîready to be sent to the aggregation server for validation and blockchain recording.

2. Validating Camera Authenticity

Click to view full size - Note: "Calibration file" refers to the camera's unique sensor identifier

Privacy-Preserving Validation

The manufacturer confirms that images come from real cameras without being able to see what photographers are capturing or track individual cameras.

  • Encrypted camera identifiers: Each camera's unique sensor identifier is encrypted using randomly assigned keys, so only the manufacturer can verify it
  • Separation of concerns: Manufacturers receive the encrypted camera ID but never see image hashes, timestamps, or locations
  • Anonymity sets: About 12,000 cameras share each encryption key, preventing tracking of individual cameras
  • Binary response: Manufacturer returns only PASS (legitimate camera) or FAIL (unknown/forged)‚Äînothing else

3. Verifying Images on Blockchain

Immutable Public Verification

Anyone can verify an image's authenticity by checking the blockchain record‚Äîno corporate gatekeepers or subscription services required.

  • Direct hash storage: Full SHA-256 image hashes are stored directly on-chain on the custom Birthmark blockchain, eliminating the need for complex cryptographic proofs
  • Institutional validators: The blockchain is operated by trusted institutions (universities, archives, journalism organizations) running validator nodes
  • Zero gas fees: No transaction costs‚Äîinstitutions donate hosting as a public service for truth in media
  • Public verification API: Open protocol allows anyone to build verification tools without asking permission

4. Modification Levels & Editing Workflow

Level 0: Validated Raw

Unprocessed sensor data from verified camera hardware. This is the raw output directly from the camera sensor before any processing.

Level 1: Validated

Level 2: Modified

Significant modifications (content-aware fill, cloning, compositing). Still traces back to authenticated original through provenance chain.

Editing Provenance

When photographers edit authenticated images in validated software (like Adobe Lightroom), the editing software can submit the edited version with a link to the parent image.

  • Software validation: Editing software is validated by the Software Authority, proving it's legitimate software (not which edits were made)
  • Provenance chain: Each edit creates a new blockchain record linking the edited image back to its authenticated parent
  • Transparency: Verification shows the complete editing history: "Validated Raw ‚Üí Edited in Lightroom ‚Üí Modified in Photoshop"
  • GPS opt-in: Location and capture time hashing is disabled by default‚Äîphotographers must explicitly enable it for privacy protection

Media Validation Network & Governance

Coalition-Operated Blockchain

The Birthmark Media Registry is not controlled by a single company or government. It's operated by the Media Validation Network‚Äîa coalition of journalism, fact-checking, and press freedom organizations.

Coalition Members (Target Organizations):

  • National Press Photographers Association (NPPA)
  • International Fact-Checking Network (IFCN/Poynter)
  • Committee to Protect Journalists (CPJ)
  • Bellingcat (open-source intelligence investigation)
  • Academic journalism schools
  • Regional press freedom organizations

Ineligible: News outlets (Reuters, AP, BBC) and content creators are excluded due to conflict of interest. Coalition Members must be organizations that analyze or validate media, not produce it.

Node Ownership Model

Organizations purchase validator node slots (one-time fee) that give them:

  • Validator authority: Right to operate a node that participates in network consensus
  • Governance rights: One vote per node in coalition governance decisions
  • No ownership limits: Maximum 2 nodes or 10% of total nodes per organization
  • Operational costs: ~$15-30/month VPS hosting per node (no ongoing fees to Foundation)

Governance Structure:

Phase 1 (Development): Foundation builds and validates the Registry technology

Phase 2 (Production): Node sales begin, Foundation retains voting authority for 1 year after first sale

Phase 3 (Coalition Governance): Full coalition governance with supermajority (67%) required for major decisions

Legal Resilience: If a government tries to compel a coalition member to violate the charter, that member is removed from the network. Governments can only remove individual nodes‚Äîthey cannot subvert the network itself.
